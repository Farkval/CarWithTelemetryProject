## Добавление новых типов роботов или редактирование старых:
Assets/Resources/Vehicles - префабы существующих роботов (можно добавлять/редактировать)

При добавлении/редактировании робота учитывать:
- При создании робота учитывать систему координат, рекомендуется строить префаб так: `z` - перед робота, `y` - верх.

- На префабе робота должны обязательно быть закреплены скрипты:
`PlayerIdentifer.cs`, `PythonNetRunner.cs`

- Можно реализовать и закрепить за префабом собственный класс логики движения робота, создаваемый класс должен реализовывать интерфейс `IRobotApi.cs` (Существующие решения: `TrackedTankController.cs`, `FourWheelsCarController.cs`)

- Можно добавлять датчики на робота: компас, gps и энкодеры реализуются собственноручно в классе логики робота. Лидары и камеры рекомендуется навешивать на EmptyObject(z - вектор-направление лидара/камеры) путем добавления скриптов: `FlashLidar.cs`, `MechanicalLidar.cs`, `MemsLidar.cs`, `CameraSensor.cs`. Для визуализации рекомендуется добавлять скрипты рядом: `LidarVisualizer.cs`, `CameraVisualizer.cs`. 

- Если пишете собственный контроллер управления роботом: "прописать" его в методах `GarageController.ChangeCarLogicEnabled()` и `GarageController.GatherComponents()` :), да, хардкод

## Редактирование карты
Редактирование и создание карты осуществляется в симуляторе через отдельную сцену. Сохранение происходит в файл .json.

## Программирование робота
Програмный код для управления роботом пишется в .py файле, например
```python
def update(robot, dt):
    pass
```
`update` - вызывается симулятором каждый кадр, robot - реализация интерфейса `IRobotApi`, пример кода режит в папке `Examples`

## Симуляция и отладка
Играть -> Загрузить карты -> Добавить, выбрать игрока -> Выбрать точку старта -> Выбрать робота -> Загрузить робота -> Выбрать скрипт управления (.py файл) -> Старт
